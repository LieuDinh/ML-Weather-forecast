# ML Weather Forecast

Weather forecasting project for Vietnam rainfall prediction. The repository contains:

- A machine learning pipeline that cleans weather data, engineers features, trains/evaluates models, and exports prediction artifacts.
- A FastAPI backend that serves precomputed XGBoost rainfall predictions.
- A React + Vite frontend dashboard for charts, lookup, comparison, and paginated prediction data.

## Project Structure

```text
.
|-- main.py                         # Recommended ML pipeline entrypoint
|-- pipeline_v3.py                  # Core preprocessing, training, and evaluation logic
|-- requirements.txt                # Python dependencies
|-- api/
|   |-- main.py                     # FastAPI app entrypoint: api.main:app
|   |-- store.py                    # Loads CSV prediction data into memory
|   `-- schemas.py                  # API response models
|-- frontend/                       # React + Vite app
|-- dataset/
|   |-- init/weather.csv            # Raw weather dataset
|   `-- province_region_code_mapping.csv
|-- input/                          # Preprocessed train/val/test files generated by pipeline
`-- output/                         # Timestamped model outputs and prediction reports
```

## Run the ML Pipeline

The recommended entrypoint is `main.py`:

```bash
source .venv/bin/activate
python main.py
```

This pipeline:

1. Loads raw data from `dataset/init/weather.csv`.
2. Applies cleaning and feature engineering.
3. Splits data into train, validation, and test sets.
4. Saves preprocessed artifacts into `input/`.
5. Trains/evaluates XGBoost, Random Forest, Prophet, and LSTM/GRU.
6. Writes a timestamped output folder under `output/`.

Expected generated files include:

```text
input/weather_train_2009_2018.csv
input/weather_val_2019_2020.csv
input/weather_test_2021.csv
input/scalers.pkl

output/<timestamp>/comparison_metrics.csv
output/<timestamp>/comparison_report.md
output/<timestamp>/evaluation_metrics.json
output/<timestamp>/evaluation_report.md
output/<timestamp>/predictions_all_models_2021.csv
output/<timestamp>/predictions_xgboost_2021.csv
output/<timestamp>/model_performance_comparison.png
output/<timestamp>/feature_importance.png
```

There is also an older direct pipeline entrypoint:

```bash
python pipeline_v3.py
```

Use `main.py` for the full current workflow and richer output set.

## Backend Setup and Run

The backend is a FastAPI service. It loads prediction CSV data at startup and exposes endpoints used by the frontend.

Start the backend from the repository root:

```bash
source .venv/bin/activate
uvicorn api.main:app --host 0.0.0.0 --port 8000 --reload
```

Open:

- API health check: <http://localhost:8000/health>
- Swagger UI: <http://localhost:8000/docs>
- ReDoc: <http://localhost:8000/redoc>

Quick API test:

```bash
curl "http://localhost:8000/health"
curl "http://localhost:8000/predictions?page=1&limit=5"
curl "http://localhost:8000/predictions/compare?date=2021-01-01&province_code=10"
```

### Backend Data Source

By default, `api/store.py` reads:

```text
output/20260403_084235/predictions_all_models_2021.csv
dataset/province_region_code_mapping.csv
input/weather_test_2021.csv
```

If you run the ML pipeline and want the API to serve the newest prediction file, update the default source path in `api/store.py`:

```python
DEFAULT_SOURCE_PATH = Path("output/<timestamp>/predictions_all_models_2021.csv")
```

Replace `<timestamp>` with the output directory created by `python main.py`.

## Environment and Configuration

### Python virtual environment

Always activate the Python virtual environment before running the ML pipeline or backend:

```bash
source .venv/bin/activate
```

### Backend host and port

The backend does not require a `.env` file. Configure host and port through the `uvicorn` command:

```bash
uvicorn api.main:app --host 127.0.0.1 --port 8000 --reload
```

For local frontend development, keep the backend on port `8000` unless you also update the frontend API base URL.

### Frontend API URL

The frontend currently calls the backend at:

```javascript
http://localhost:8000
```

This value is defined in:

```text
frontend/src/api/weatherApi.js
```

If the backend runs on another host or port, update the `baseURL` there before starting or rebuilding the frontend.

## Troubleshooting

### `ModuleNotFoundError`

Activate the virtual environment and reinstall dependencies:

```bash
source .venv/bin/activate
pip install -r requirements.txt
```

### API returns `FileNotFoundError` on startup

Check that the prediction CSV configured in `api/store.py` exists. If you generated a new output folder, update `DEFAULT_SOURCE_PATH` to point to the new `predictions_all_models_2021.csv`.

### Frontend cannot load data

Confirm the backend is running:

```bash
curl http://localhost:8000/health
```

If the backend is running on a different port, update `frontend/src/api/weatherApi.js`.

### Prophet installation issues

`prophet` can require extra build dependencies depending on Python version and OS. If installation fails, try a stable Python version such as Python 3.10 or 3.11, recreate the virtual environment, and reinstall `requirements.txt`.
